> 版本：v1.0

## 1. 视觉设计规范

### 1.1 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主背景 | 淡蓝色 | `#D4E8F7` |
| 窗口背景 | 更浅淡蓝 | `#EAF2FA` |
| 组件背景 | 白色 | `#FFFFFF` |
| 主文字 | 深灰/黑 | `#2C3E50` |
| 辅助文字 | 中灰色 | `#7F8C8D` |
| 主按钮（开始） | 浅绿色 | `#27AE60` |
| 停止按钮 | 浅红色 | `#E74C3C` |
| 边框 | 淡灰蓝 | `#BDC3C7` |

### 1.2 窗口规格

- 窗口尺寸：固定 `350 x 220` 像素，不可拉伸
- 窗口标题：`AutoClicker`
- 窗口置顶：可选（暂不实现）

### 1.3 布局结构

```
┌─────────────────────────────────┐
│  AutoClicker                    │  ← 标题栏
├─────────────────────────────────┤
│                                 │
│  选择连点键:  [捕获按键 ▼]      │  ← 键盘捕获或鼠标下拉
│                                 │
│  时间间隔(ms): [ 100  ]         │  ← 数值输入框
│                                 │
│       [ 开始 ]  [ 停止 ]        │  ← 控制按钮
│                                 │
│  状态: 空闲 / 运行中            │  ← 状态提示
│                                 │
└─────────────────────────────────┘
```

### 1.4 字体

- 字体：Microsoft YaHei / Segoe UI（系统默认）
- 字号：正文 10pt，标题 12pt

## 2. 交互设计规范

### 2.1 键选择交互

- **键盘键**：点击「捕获按键」按钮 → 按钮文字变为「按下键盘任意键...」→ 用户按下键盘键 → 按钮恢复并显示已捕获的键名
- **鼠标键**：下拉框直接选择左键/右键/中键/侧键X1/侧键X2

### 2.2 开始/停止逻辑

- **开始按钮**：点击后按钮变为灰色（不可重复点击），停止按钮可用
- **停止按钮**：灰色（不可点击）→ 点击开始后变为可用 → 点击停止后恢复灰色
- 软件启动时：停止按钮默认灰色不可点击

### 2.3 运行中状态

- 窗口标题栏追加 `(运行中...)`
- 状态标签实时显示当前状态
- 关闭窗口时自动停止连点

## 3. 代码设计规范

### 3.1 文件职责

| 文件 | 职责 |
|------|------|
| `main.py` | 入口，初始化并启动 GUI |
| `gui.py` | 界面构建、事件绑定、用户交互 |
| `clicker.py` | 连点核心引擎：线程管理、键盘/鼠标模拟 |
| `config.py` | 配置常量、默认值 |

### 3.2 编码规范

- 使用英文变量名和注释
- 函数名采用 snake_case
- 类名采用 PascalCase
- 单文件不超过 300 行
- 关键逻辑需写注释说明

### 3.3 安全与异常处理

- 开始连点前校验间隔值是否合法（正整数，≥ 10ms）
- 捕获异常不导致程序崩溃
- 程序退出时确保连点线程正确终止
