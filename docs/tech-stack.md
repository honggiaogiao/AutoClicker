# 连点器 — 技术选型文档

> 版本：v1.0  
> 最后更新：2026-05-24

---

## 1. 技术选型

### 编程语言：Python 3.x

**理由：**
- 跨平台兼容，开发效率高
- 生态丰富，GUI、系统控制、打包工具成熟
- 代码可读性强，便于后续维护

### GUI 框架：tkinter

**理由：**
- Python 标准库自带，零额外依赖
- 足够满足简洁界面的需求
- 打包后体积小

### 键盘/鼠标控制库

| 用途 | 库 | 说明 |
|------|-----|------|
| 键盘全局控制 | `keyboard` | 支持全局按键监听、模拟按键，后台生效 |
| 鼠标控制 | `mouse` | 支持全局鼠标点击、位置控制 |

**理由：**
- `keyboard` 和 `mouse` 库在 Windows 下通过底层 API 实现，无需管理员权限即可运行
- 支持后台运行（窗口不处于前台时仍生效）

### 打包工具：PyInstaller

**理由：**
- 将 Python 脚本打包为单 exe 文件
- 支持 Windows 平台
- 用户双击即用，无需安装 Python

## 2. 依赖清单

```
python >= 3.8
tkinter      # Python 标准库，无需额外安装
keyboard     # 键盘全局控制
mouse        # 鼠标全局控制
pyinstaller  # 打包工具（仅开发环境需要）
```

## 4. 为什么不选其他方案

| 方案 | 放弃理由 |
|------|----------|
| C# WinForms | 需要 VS 开发环境，开发和调试门槛高 |
| Electron | 打包体积过大（>50MB），杀鸡用牛刀 |
| AutoHotkey | 功能足够但 UI 开发不友好，不符合简洁直观的要求 |
| C++ Win32 API | 开发周期长，不适合快速迭代 |
